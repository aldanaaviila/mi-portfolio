# Aldana Avila — Portfolio

Portfolio personal de Data Analyst. Sitio web con mi presentación profesional, stack técnico, proyectos aplicados y línea de tiempo de mi trayectoria.

🔗 **Ver sitio en vivo:** https://github.com/aldanaaviila/mi-portfolio

## Sobre el proyecto

Sitio estático construido desde cero, sin frameworks, para mostrar mi experiencia como Analista de Datos, Logística y RRHH en DELTACOM SA, y mi formación en Desarrollo de Software (UADE).

## Tech stack

- **HTML5** — estructura semántica
- **CSS3** — diseño responsive, variables CSS, animaciones con `IntersectionObserver`
- **JavaScript (vanilla)** — filtros de proyectos, modal de detalle, menú mobile
- Tipografías: [DM Sans](https://fonts.google.com/specimen/DM+Sans) y [DM Serif Display](https://fonts.google.com/specimen/DM+Serif+Display) (Google Fonts)

## Estructura del proyecto

```
├── index.html              # Estructura de todas las secciones
├── style.css                # Estilos (variables, secciones, responsive)
├── script.js                # Interactividad (nav, filtros, modal, animaciones)
├── dashboard-logistica.png  # Captura del dashboard de Logística
├── dashboard-rrhh.png       # Captura del dashboard de RRHH
├── dashboard-finanzas.png   # Captura del dashboard de Finanzas Personales
├── dashboard-empanadas.png  # Captura del dashboard de Empamaris
└── README.md
```

## Secciones

- **Hero** — presentación e introducción
- **Sobre mí** — resumen profesional y formación
- **Skills** — stack técnico por categoría (Data & BI, Bases de datos, Programación, Herramientas, Idiomas, Formación)
- **Proyectos** — tarjetas filtrables por categoría (BI & Dashboards, SQL & BD, Automatización), con modal de detalle por proyecto
- **Mi camino en datos** — línea de tiempo de mi trayectoria laboral y académica
- **Contacto** — Email, LinkedIn y WhatsApp

## Cómo verlo localmente

No requiere instalación ni build. Solo cloná el repo y abrí `index.html` en el navegador:

```bash
git clone https://github.com/TU-USUARIO/TU-REPO.git
cd TU-REPO
```

Luego abrí `index.html` con doble clic, o con la extensión **Live Server** de VS Code para recarga automática al editar.
